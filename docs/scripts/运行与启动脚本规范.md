# 运行与启动脚本规范

## 1. 环境

- OS: CentOS 8
- Python: 3.12
- Conda env: `aishopping-py312`

## 2. 标准流程

```bash
bash scripts/setup_conda_env.sh
source ~/miniconda3/etc/profile.d/conda.sh
conda activate aishopping-py312
bash scripts/start_product_module.sh
bash scripts/start_user_module.sh
bash scripts/start_agent_module.sh
bash scripts/start_web.sh
```

## 3. 端口与密钥

- Web 端口：`6008`
- `TAVILY_API_KEY`
- `DASHSCOPE_API_KEY`

## 4. 启动脚本职责建议

1. `start_product_module.sh`
   - 加载离线统计产物
   - 启动 product_understanding 服务
2. `start_user_module.sh`
   - 启动用户画像 mock 服务
3. `start_agent_module.sh`
   - 启动 Orchestrator + Assistant API
4. `start_web.sh`
   - 启动前端（指向 6008）

## 5. 健康检查

每个脚本启动后应输出：

- PID
- 监听端口
- 日志文件路径
- 最近一次配置加载状态
